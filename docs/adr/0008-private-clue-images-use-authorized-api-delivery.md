# Private clue images use authorized API delivery

Campaign Clue images live in private S3-compatible storage, using Cloudflare R2 in production and MinIO locally, and are served through the API only after the clue's visibility is authorized. Uploads are decoded, validated, reoriented, stripped of metadata, and safely re-encoded before only the normalized image is stored. Public and presigned object URLs are rejected so access can be revoked immediately; clients instead use private HTTP revalidation with stable ETags, allowing authorized repeat requests to receive `304 Not Modified` without caching image bytes in Redis. This accepts API bandwidth in exchange for strict authorization, while leaving metadata caching as a future optimization only if measurements justify it.
